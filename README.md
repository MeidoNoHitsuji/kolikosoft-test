# Веб сервис получения цен из skinport

## Как запустить сервис?
1. Создать .env файл на основе .env.example
2. Запустить `docker compose up -d --build`

## ENV переменные
Все необходимые ENV переменные указаны в .env.example. Но вот инстукция по ним:
```
POSTGRES_HOST - хост базы Postgres
POSTGRES_PORT - порт базы Postgres
POSTGRES_USER - имя пользователя базы Postgres
POSTGRES_PASSWORD - пароль пользователя базы Postgres
POSTGRES_DATABASE - имя базы базы Postgres
POSTGRES_SSL_MODE - ssl mode

REDIS_HOST - хост Redis
REDIS_PORT - порт Redis
REDIS_PASSWORD - пароль от Redis (если нужно, но можно оставить пустым)
REDIS_DATABASE - id базы Redis (лучше оставить 0)
```

## Роуты, которые есть (хотя это можно было заменить swagger'ом)
1. /api/v1/account/:id - получение информации пользователя (дефолтно создаётся id = 1 пользователь)
2. /api/v1/account/:id/history - получение истории пользователя
3. /api/v1/account/:id/add-money - добавить денег пользователя (принимает json с параметром value)
4. /api/v1/account/:id/buy-item - покупка предмета пользователем (принимает json с параметром hash_market_name)
5. /api/v1/items - возвращает весь список предметов

## Ещё немного информации
1. Цены обновляются раз в 5 минут. Хотя даже с таким лимитом (который буквально указан в документации api) я поймал rate-limit. Но и на такой случай есть защита, что лимитёр будет ждать отведённое ему время, полученное из "Retry-After", если таковое пришло.
2. Не у всех предметов есть цена.
3. В ./config/application.json можно поменять url api skinport, порт на котором запускается сервис внутри контейнера, а также уровень логов (в идеале сюда бы ещё трейсы намутить, но я решил обойтись без них)
4. В docker-compose.yml создаётся дефолтная база со своими данными. Если вы поменяли данные там, то нужно поменять их и в env файле. Тоже самое касается и порта запускаемого приложения.
5. Приложение можно запустить и вне docker'а. Просто выполните `make build && ./app` и оно должно запустить.
6. Сбилженное приложение принимает флаг "config", которое указывает пусть до application.json. Дефолтно оно обращается к пути "./config/application.json", но если вам нужно, то его можно поменять.
